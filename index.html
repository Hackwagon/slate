    Introduction

    Welcome to the Hackwagon API! You can use our API to access Hackwagon API endpoints, which currently only allows our approved partners to access certain protected endpoints. More to come soon.

    We have language bindings in REST for now only, and more to come. You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    The API endpoint for Hackwagon's API follows the following structure: https://api.hackwagon.com/v1/

    Authentication

    Example API call with embedded API key:

    fetch('https://api.hackwagon.com/v1/example', {
  method: "POST",
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'sk_superawesomesecretkey',
    'X-App-Id': 'randomappid'
  },
  body: JSON.stringify({
    key: 'value'
  })
}).then(function(response) {
  return response.json()
}).catch(function(ex) {
  console.log('parsing failed', ex)
})

    import requests

url = "https://api.hackwagon.com/v1/example"
headers = { 'Content-Type': 'application/json', 'X-API-Key': 'sk_superawesomesecretkey', 'X-App-Id': 'randomappid' }
r = requests.get(url, headers=headers)
data = r.json()

    Make sure to replace sk_superawesomesecretkey with your API key, and randomappid with your Application id.

    Hackwagon uses API keys and Application Ids to allow access to the API. You can register a new Hackwagon API key by contacting us at hello@hackwagon.com.

    Hackwagon expects for the API key and Application Id to be included in all API requests to the server in the header that looks like the following:

    X-API-Key: sk_superawesomesecretkey

    X-App-Id: randomappid

    From here on it is expected that all requests contain the API key and application Id header.

    You must replace sk_superawesomesecretkey with your personal API key, and randomappid with your personal Application Id.

    Queue System

    Appending Queue Number

    Example API call:

    fetch('https://api.hackwagon.com/v1/company_queue', {
  method: "POST",
  body: JSON.stringify({
    merchantId: 'value',
    number: '010',
  })
}).then(function(response) {
  return response.json()
}).catch(function(ex) {
  console.log('parsing failed', ex)
})

    import requests

url = "https://api.hackwagon.com/v1/company_queue"
headers = { 'Content-Type': 'application/json', 'X-API-Key': 'sk_superawesomesecretkey', 'X-App-Id': 'randomappid' }
r = requests.post(url, headers=headers)
data = r.json()

    Request Body:

    {
  "merchantId": "string",
  "number": "string"
}

    Expected response structure:

    {
  "status": "success|failed",
  "message": "A message detailing the success or error message"
}

    This endpoint appends a new queue number to be displayed in our physical devices located around Singapore. By specifying a merchantId, you'll be able to target a specific device. The number is the queue number you wish to display on the device (try to keep it to 3 characters or less).

    HTTP Request

    POST http://api.hackwagon.com/v1/company_queue

    Query Parameters

    Parameter Expects Description
    merchantId string This is the Id you assign to your own merchants in your own applications. This key will be stored by us, so your application can identify the physical device.
    number string This number is supposed to be taken in as a string. Keep it to 3 characters or less.
    The merchantId must be provided to Hackwagon's team so we can assign it to each unique physical device for your identification.

    Response Parameters

    Parameter Possible Value(s) Description
    status success,failed This describes whether your API call was a success or failure
    message string This is a success/error description

    This endpoint dumps all the data in our registration system. By specifying an API key with appropriate permissions, you'll be able to receive a dump of the data. NRIC of students has been redacted for the sake of privacy.

    HTTP Request

    GET http://api.hackwagon.com/v1/registrations/dump

    Response Parameters

    Parameter Possible Value(s) Description
    _id random_id Unique id for the record
    userId random_id User that registered under this registration record
    classId random_id Unique class id, could be either DS101 classes or DS102 classes
    status pending-student-verification,pending-manual-verification,slot-confirmed,slot-rejected,student-slot-confirmed,slot-review Could be any one of the possible values
    studentDetails object An object detailing the student's details. Details should be self explanatory
    createdAt date A JavaScript date object detailing when the student created the registration record (sign up date)
    updatedAt date A JavaScript date object detailing when the registration record was last updated,
    deposit boolean Did the student confirm their slot by placing a $100 deposit
    formId string An id to identify the student's uploaded form 1 or form 2 file (optional)
    matriculationLetterId string An id to identify the student's uploaded matriculation letter (optional)
    elevenBId string An id to identify the student's uploaded eleven B (optional)
    serviceTranscriptId string An id to identify the student's uploaded service transcript (optional)
    citrepId string An id to identify the student's citrep registration id

    This endpoint dumps all the data in our class system. By specifying an API key with appropriate permissions, you'll be able to receive a dump of the data.

    HTTP Request

    GET http://api.hackwagon.com/v1/classes/dump

    Response Parameters

    Parameter Possible Value(s) Description
    _id random_id Unique id for the class
    courseId random_id Unique course id, could be either DS101 classes or DS102 classes
    archive boolean Determines whether the class is archived, and visible for viewing
    hoursPerLesson integer How many hours does the class go on for
    classDates [ date ] An array of dates for each lesson in the class
    createdAt date A JavaScript date object detailing when the class was created
    updatedAt date A JavaScript date object detailing when the class record was last updated
    openForRegistration boolean Is the class still currently open for registration?
    forTier students,adults,both Which type of students is the class open for
    classList [ random_id ] An array of user ids for the students that are in the class
    instructors [ random_id ] An array of user ids for the instructors that are in the class
    teachingAssistants [ random_id ] An array of user ids for the teaching assistants that are in the class
    attendanceList object Used to store the attendance data
    attendanceList.userId object Used to store the attendance data for a user, based on his/her user id under userId
    ..userId.0 object To retrieve a student's attendance on a particular day where 0 could be any number based on the number of lessons in classDates (length of classDates)
    ..userId.0 object To retrieve a student's attendance on a particular day
    ..userId.0.status attended,absent,substitute To retrieve a student's attendance status on a particular day
    ..userId.0.classId random_id If student is on make up, a classId-lessonNum will be assigned

    Errors

    The Hackwagon API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- You do not have sufficient permissions to query this endpoint.
    404 Not Found -- The specified API endpoint could not be found.
    405 Method Not Allowed -- You tried to access an API with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The API requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many API endpoints! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
